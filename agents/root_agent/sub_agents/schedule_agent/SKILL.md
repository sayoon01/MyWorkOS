# 절차: 일정 등록 요청 처리

0. 사용자 발화에 반복 적용되는 선호·제약이 있는지 가장 먼저 확인한다.
   트리거 예시: "항상 ~로 해줘", "앞으로는/다음부터는 ~", "매번 ~",
   "오후 2시 이후로만", "금요일은 회의 금지", "항상 화상으로"
   → 해당되면 다른 단계보다 먼저 save_memory(note="일정 선호: <구체적 내용>")를 호출한다.
   → 제목/시간 등 다른 정보가 부족해 결국 되물어야 하는 경우에도, save_memory 호출은 예외 없이 먼저 한다.

1. get_current_time 호출 → 상대적 날짜 표현을 절대 날짜로 변환
2. 제목/종료시각(또는 소요시간)/참석자 중 빠진 정보 확인 → 없으면 되묻고 종료
   (create_event 등 등록 도구는 호출하지 않는다. 단 0번의 save_memory는 이미 실행된 상태여야 한다)
3. list_events(date)로 같은 날 기존 일정 조회 → 시간 겹치면 경고
4. 정보가 모두 확보되면 create_event 호출.
   이때 0번 또는 load_memory로 확인한 과거 선호(예: "오후 2시 이후만")가 있으면 그 기준으로 먼저 검증한다.
5. 참석자가 전원 확정 안 됐으면 결과에 "(가일정)" 표시
6. 사용자가 명시적으로 요청한 경우에만 set_reminder 호출
